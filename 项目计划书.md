## 一、项目概述

### 1. 项目背景
校园快递量日益增多，许多学生因上课、实习等原因无法及时取件，同时存在有闲暇时间的学生愿意通过跑腿赚取少量报酬。本系统旨在搭建一个校园内的快递代取信息平台，让发布任务与接单更加便捷、规范。

### 2. 项目目标
实现一个快递代取跑腿管理系统，支持学生发布取件订单、跑腿员浏览并接单、订单状态实时跟踪（待接单→已接单→配送中→已完成），并提供个人订单管理功能。

### 3. 开发环境与条件
- 开发工具：VS Code / IntelliJ IDEA
- 数据库：mysql
- 运行环境：JDK 17 + MySQL 8.0 + Windows

---

## 二、需求分析

### 1. 目标用户与使用场景
面向对象：
- **普通用户（学生）**：需要代取快递，发布订单；查看自己发布的订单状态；对跑腿员进行评价。
- **跑腿员用户**：浏览未接订单，接单，更新配送进度，确认完成订单，查看累计收入与完成单数。
使用场景：校园内各快递点→宿舍楼。

### 2. 核心功能清单
1. **用户模块**：注册、登录、角色区分（普通用户 / 跑腿员）
2. **订单模块**：发布订单、接单、更新进度、确认完成、取消订单
3. **评价模块**：订单完成后对跑腿员进行评分和留言
4. **结算模块**：跑腿员查看累计收入与完成单数

### 3. 非功能需求
- **性能要求**：普通页面响应时间≤2秒，并发用户数≤50。
- **安全要求**：用户密码加密存储（如BCrypt），防止简单SQL注入。
- **兼容性要求**：建议使用Chrome浏览器访问。

## 三、技术路线

### 1. 总体架构
采用前后端分离架构，前端通过AJAX调用后端RESTful API，后端连接MySQL数据库。

### 2. 主要技术
- **后端**：Java 17 + Spring Boot 2.7.x + MyBatis-Plus
- **前端**：Vue3 + Vite + Element Plus + Vue Router + Axios + Pinia
- **数据库**：MySQL 8.0

### 3. 数据设计初步考虑
主要实体：用户（User）、订单（Order）、评价（Review）、结算记录（Settlement）。  
- **用户表**：用户ID、姓名、学号、角色（普通用户/跑腿员）、密码（加密）、手机号、跑腿员累计收入、完成单数等  
- **订单表**：订单ID、发布者ID、接单者ID、取件码、快递点、宿舍楼、状态（0待接单/1已接单/2配送中/3已完成/4已取消）、发布时间、完成时间等  
- **评价表**：评价ID、订单ID、评分（1-5）、留言内容、评价时间  
- **结算记录表**：用于记录每次完成订单后跑腿员的收入变动
